Guía GitHub ADSO

Estructura GitHub Ramas GitHub Pages Evaluación
1. Estructura del proyecto según la imagen

El proyecto tiene una estructura organizada para un sitio web de varias páginas:

taller3/ ├── app/ │ ├── forms.html │ ├── tables.html │ └── github.html ├── assets/ │ ├── css/ │ ├── img/ │ └── js/ ├── index.html └── README.md
2. Requisitos iniciales
  • Tener una cuenta activa en GitHub.
  • Tener instalado Git en el computador.
  • Tener instalado Visual Studio Code.
  • Abrir la carpeta del proyecto desde Visual Studio Code.
3. Configuración de Git desde Visual Studio Code

Abre la terminal integrada de Visual Studio Code:

Menú: Terminal → New Terminal

Configura tu nombre y correo:

git config --global user.name "Tu Nombre" git config --global user.email "tu_correo@email.com"

Verifica la configuración:

git config --global --list
4. Inicializar Git en el proyecto

Ubícate en la carpeta raíz del proyecto, donde está el archivo index.html, y ejecuta:

git init git add . git commit -m "Primer commit: estructura inicial del proyecto"
Este primer commit guarda la versión inicial del proyecto, incluyendo index.html, forms.html, tables.html y la carpeta assets.
5. Crear repositorio en GitHub
  1. Ingresar a GitHub.
  2. Hacer clic en New repository.
  3. Asignar un nombre al repositorio, por ejemplo: proyecto-bootstrap-adso.
  4. Seleccionar si será público o privado.
  5. Crear el repositorio sin agregar README si el proyecto ya tiene archivos locales.
6. Conectar proyecto local con GitHub

Copia la URL del repositorio creado y ejecuta:

git remote add origin https://github.com/usuario/proyecto-bootstrap-adso.git git branch -M main git push -u origin main
Cambia usuario y proyecto-bootstrap-adso por los datos reales del repositorio.
